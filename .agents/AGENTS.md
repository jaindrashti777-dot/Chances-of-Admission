# Agent Behavioral Rules

## Strict Subsystem Workflow
When building out projects, always follow this strict 3-step workflow:

**Step 1 — Build one subsystem**
Focus exclusively on one subsystem at a time (e.g., Backend, Frontend, ML, Database). Do not try to build the entire application at once.

**Step 2 — Review**
Do not immediately continue. Instead, conduct a rigorous self-review. Act as a Staff Engineer conducting a pull request review specifically for that subsystem.

**Step 3 — Test**
Run the subsystem. Fix any bugs that arise. Commit the code. Only after this is complete may you continue to the next subsystem.
