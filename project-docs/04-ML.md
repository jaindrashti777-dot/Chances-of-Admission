# Machine Learning Pipeline Architecture

## Overview
This document outlines the machine learning pipeline designed to predict closing ranks and estimate admission probabilities.

## Pipeline Components

### 1. Data Cleaning (`ml.src.data.make_dataset`)
- Handles duplicates, missing values, and type conversions.
- Normalizes textual categories to ensure consistency across years.
- Validates the presence of `closing_rank`.

### 2. Exploratory Data Analysis (`ml.src.visualization.visualize`)
- Automated generation of distribution plots, correlation matrices, and category breakdowns.
- Visualizations are stored in `ml/reports/eda`.

### 3. Feature Engineering (`ml.src.features.build_features`)
Uses `sklearn.compose.ColumnTransformer`:
- **Categorical (OneHot)**: Institute Type, Category, Quota, Seat Pool, Counselling Body.
- **Categorical (Ordinal)**: College Name, Branch Name (handles high cardinality).
- **Numerical (Scaled)**: Year, Round Number.

### 4. Model Training (`ml.src.models.train_model`)
- Compares Baseline Linear Regression, Random Forest, and XGBoost.
- Performs GridSearchCV to tune hyperparameters (e.g., `n_estimators`, `max_depth` for XGBoost).
- Persists the best `Pipeline` (including `ColumnTransformer` + `Regressor`) to `ml/models/best_model.joblib`.
- Records metrics (MAE, RMSE, R2) to `ml/reports/metrics/training_metrics.json`.

### 5. Prediction Pipeline (`ml.src.models.predict_model`)
- Wrapper class `Predictor` loads the model artifact.
- `predict_probability` estimates a probability score based on the user's rank relative to the predicted closing rank.

### 6. Explainable AI (`ml.src.models.explain_model`)
- Uses SHAP (`shap.TreeExplainer`) to generate global feature importance summaries.
- Allows developers to inspect which features (e.g., Quota, Category) drive the predictions.

## Retraining Workflow
To retrain the model with new data (e.g., next year's cutoffs):
1. Place new raw data in `ml/data/raw/`.
2. Run `python -m ml.src.data.make_dataset`.
3. Run `python -m ml.src.models.train_model`.
4. Review new metrics and SHAP plots. The FastAPI backend will automatically load the new `.joblib` file on startup.
